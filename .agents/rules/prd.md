# Product Requirements Document (PRD): SceneIt

## 1. Overview
**Product Name:** SceneIt
**Platform:** Web, iOS, Android (Responsive Web App)
**Description:** SceneIt is a frictionless, Tinder-style movie recommendation app. Users swipe on movie posters to indicate their watch history and preferences. Based on these interactions, the app uses AI to generate highly personalized movie recommendations. No account creation is required to get a recommendation, allowing users to jump straight into the experience.

## 2. Target Audience
- Movie enthusiasts looking for their next watch.
- Casual viewers who struggle with "decision fatigue" when choosing a movie.
- Users who enjoy gamified, swipe-based interfaces.
- People that want an agnostic and streaming platform independant movie recommendation.

## 3. Core Features
### 3.1. Swipe Interface
Users are presented with a stack of movie cards. Each card displays:
- Movie Title
- Release Year
- Genre
- Director
- Synopsis
- A visually appealing gradient background overlayed on top of the movie poster.

**Swipe Actions:**
- **Swipe Left:** Unwatched (User hasn't seen the movie).
- **Swipe Right:** Watched (User has seen the movie, neutral feeling).
- **Swipe Up:** Loved (User has seen the movie and loved it).
- **Swipe Down:** Disliked (User has seen the movie and disliked it).

### 3.2. Manual Controls
For users who prefer tapping over swiping, or for accessibility, manual control buttons are provided below the card stack:
- Eye Off icon (Unwatched)
- Eye icon (Watched)
- Heart icon (Loved)
- Thumbs Down icon (Disliked)

### 3.3. AI-Powered Recommendations
- A "Recommend" button is available in the header.
- When tapped, the app aggregates the user's swipe history (Loved, Watched, Disliked, Unwatched).
- The app sends this context to the Gemini AI model to generate a single, highly personalized movie recommendation that the user has not yet seen.
- The recommendation screen displays the movie details along with a specific "Why you'll love it" reason generated by the AI.

### 3.4. Infinite Scrolling/Swiping
- The app continuously fetches new batches of movies using the Gemini AI model to ensure the user never runs out of cards to swipe.
- The fetching mechanism excludes movies the user has already seen in the current session to prevent duplicates.

### 3.5. Frictionless Onboarding
- No login, signup, or account creation is required.
- State is managed locally during the session.
- Recommendation is reset when user leaves or session is refreshed.

### 3.6. User Account for Additional Features
- Users can sign up for an account to save their recommended movies into a "Watch Later" list.
- Users' watch, loved, unwatched, and disliked history from swiping is saved to their profile and they can see their history.
- Other account features to be added later.

## 4. User Flow
1. **Launch:** User opens the app and is immediately presented with the first movie card.
2. **Interact:** User swipes cards in four directions or uses the control buttons.
3. **Feedback Loop:** As the user swipes, the app builds a temporary profile of their tastes.
4. **Recommendation Request:** User taps the "Recommend" button.
5. **Result:** User views the AI-generated recommendation and the reasoning behind it.
6. **Continue:** User can dismiss the recommendation to return to swiping and further refine their profile.

## 5. Technical Requirements
- **Frontend Framework:** Next.js (App Router), React.
- **Styling:** Tailwind CSS.
- **Animations:** Motion (Framer Motion) for fluid swipe gestures, card stacking, and UI transitions.
- **Icons:** Lucide React.
- **AI Integration:** `@google/genai` SDK using the `gemini-3-flash-preview` model for both generating the infinite feed of movies and providing the final personalized recommendation.
- **Responsiveness:** Mobile-first design ensuring touch targets are large enough (min 44px) and gestures work seamlessly on iOS and Android browsers.

## 6. Future Enhancements (V2)
- **Persistent Storage:** Save swipe history in `localStorage` so users don't lose their profile if they close the browser.
- **Real Posters:** Integrate with TMDB or OMDB API to fetch actual movie posters instead of gradient backgrounds.
- **Streaming Availability:** Show where the recommended movie is currently streaming (e.g., Netflix, Hulu, Max).
- **Shareability:** Allow users to share their AI recommendation with friends via a native share sheet or link.
- **Watchlist:** Allow users to save recommendations to a watchlist for later.
